How do I create my own templates?

In the Template tab on the options page you can click on the "Need help writing templates?" link at the bottom to view the built-in help section for writing templates. If for some crazy reason you don't yet have Template installed (can't understand why not) you can still learn about writing templates here.

Why can't I use some features on certain pages?

Due to Chrome's security implementations some features won't work on certain pages. These pages are anything on the Chrome Web Store and any URL using the chrome:// protocol as these are pages internal to the browser (including pages specific to extensions/apps). In these cases keyboard shortcuts, context (right-click) menus as well as extraction of selected text/links may not work.

Why are the changes to my templates not being saved?

On the options page be sure that you always press "Save & Close" button whenever you want to save your changes. This applies even when you are only adding, deleting, reordering and importing templates.

Why can I not edit some templates?

The title and content fields of predefined templates (URL, Short, Anchor etc.) are read-only and cannot be changed. Also, you cannot delete these templates but you can disable them like any other template to prevent them from being used.

What exactly do you mean "Supported Extensions"?

The extensions that are listed in the Supported Extensions section have treated specially by Template so that the tab URL and title can still be extracted. These extensions tend only to be those that would mask the URL and other information of the current page.

What URL is used when I right-click on a page?

Well, that absolutely depends what it is you right-clicked on. Template currently checks what you clicked on in the following priority;

  1. Frame - Right-clicking anywhere within an internal frame
  2. Link - Right-clicking on a link
  3. Media - Right-clicking on anything that has a src attribute (e.g. images)
  4. Page - Right-clicking anywhere else on the page that doesn't match any of the above

There is currently an issue raised that involves re-prioritizing this list so if you're interested take a look.

Why am I seeing this error message...?

Due to the extent of customization options available for Template it's no surprise that something could go wrong. If it does it's nice to know what that error message means;

  • "Empty templates cannot be copied" - This occurs if the template being used has either no content or the result after being parsed is empty. If you think about it, you can't exactly copy an empty string to the clipboard can you? Go on and try, I'll wait... I told you so.
  • "URL is not properly encoded" - This occurs if the user is attempting to use Template on a URL that is not properly encoded. This is due to the underlying technology used for extract the information on each URL (e.g. domain, port, parameters).
  • "An error occurred when collecting page information" - This occurs when an unexpected error was thrown when building up the template data. Please can you raise an issue while providing as much information as possible so I can investigate and fix it.
  • "Template could not be copied" - This occurs when an unexpected error was thrown during the copy process. Please can you raise an issue while providing as much information as possible so I can investigate and fix it.
  • "Configuration problem found with <URL Shortener>" - This occurs if the active URL shortener requires a target URL to be specified and this has not been done (e.g. YOURLS).
  • "Could not get a successful response from <URL Shortener>" - This occurs if the Template is unable to successfully get a response from the active URL shortener. This can be for multiple reasons but you should check you configuration for that service first.

What happened to "URL Copy"?

This extension was previously named "URL Copy" but was renamed in v0.2.0.0 as I felt it had evolved far beyond that and the name no longer did it justice. "Template" provides a much more accurate description of the functionality and power of this extension.

Why isn't Chrome updating Template?

Chrome's update frequency is slightly unpredictable (at least in my eyes) and I've found that sometimes it updates my extensions as soon as they're updated but I've also seen it some extensions not getting updated even when using the browser for hours. However, there is a little-known way (at least to non-developers) to force it to update your extensions;

  • Open the Extensions page via on of the following methods;
    • Main Menu: Tools > Extensions
    • Address Bar: chrome://extensions
    • Right-click Template's toolbar icon and select "Manage extensions..."
  • Click on "Developer mode" in the top right of the page so that it has a collapse (-) icon on its left.
  • Click the "Update extensions now" button, et viola!

Your extensions should now be receiving updates (where available) and you may notice some extensions turning off and on again. If an extension requires any additional permissions that you have not already authorized it will not be turned back on until you have granted these permissions.
