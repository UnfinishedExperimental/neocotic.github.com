Lists & Objects are generally accessed using sections (including inverted) to iterate over and access properties respectively. However, simple tags can be used to output the contents of a list or the properties of an object in CSV format.

Definitions
Name Type Description
cookies List The names of all cookies associated with the page
coordsNew Object The properties of your current physical location (e.g. latitude, longitude and many more)
fparams Object The key/value pairs of all parameters within the hash fragment (e.g. fragment foo=bar&fu=baz contains foo and fu)
fsegments List The segments from the hash fragment's path
images*New List The URLs of all images on the page
keywords*New List The keywords from the page's meta information
links*New List The URLs of all links on the page
params Object The key/value pairs of all parameters (e.g. query foo=bar&fu=baz contains foo and fu)
pluginsNew List The names of all enabled browser plugins (e.g. Shockwave Flash)
popularNew Object The properties of your most popular template (e.g. title, content, shortcut and many more)
scripts*New List The URLs of all external script used by the page
segments List The segments from the URL's path
selectedImages*New List The URLs of all images within the current selection
selectedLinks*New List The URLs of all links within the current selection
styleSheets*New List The URLs of all external CSS resources used by the page
tabsNew List The URLs of all tabs open in the current window
templateNew Object The properties of the activated template (e.g. title, image, usage and many more)
* Not available on internal Chrome pages or the Chrome Web Store

Examples

This page's information will be used in the following examples.

As it's a list, you can do the following to iterate over all the segments;

{% highlight html %} Follow the breadcrumbs; {#segments} * {.} {/segments} {% endhighlight %}

Would output the following;

{% highlight html %} Follow the breadcrumbs; * template * guide * lists-objects {% endhighlight %}

In order to access the properties of an object it couldn't be any easier than this;

{% highlight html %} {#params}foo is "{foo}" and fu is "{fu}".{/params} {^params}No parameters could be found!{/params} {% endhighlight %}

Since this page has (or at least should have) no parameters it would output the following;

{% highlight html %} No parameters could be found! {% endhighlight %}

However, if it was to have something like foo=bar&fu=baz as its query string it would output this;

{% highlight html %} foo is "bar" and fu is "baz". {% endhighlight %}

Alternatively, this could be achieved using dot notation as follows;

{% highlight html %} foo is "{params.foo}" and fu is "{params.fu}". {% endhighlight %}

Using the following you can access the name and values of all available cookies;

{% highlight html %} The site has following information stored; {#cookies} * {.} = "{#cookie}{.}{/cookie}" {/cookies} {% endhighlight %}

This will output in something like the following;

{% highlight html %} The site has following information stored; * JSESSIONID = "2pkffy9ivmpfwntf12y6o4cgu" * __utmc = "224978569" {% endhighlight %}

If you would like to embed a map of your current location (e.g. in a blog post) try the following snippet;

{% highlight html %} {#coords}
View Larger Map {/coords} {% endhighlight %}
